Core reactivity

Signals & Computed

A signal is a value that components subscribe to automatically when they read it. No watchers. No dependency lists. Just call the getter inside a render.

signal() — counter LIVE 
const [count, setCount] = signal(0);

// Getter call inside render = subscription
return () => html`
  <div>
    <p>${count()}</p>
    <button onclick=${increment}>+</button>
  </div>
`;
computed() — derived state LIVE 
const [n, setN] = signal(4);

// Re-computes only when n() changes
const squared  = computed(() => n() ** 2);
const isEven   = computed(() => n() % 2 === 0);
Reactive text binding LIVE 
const [name, setName] = signal('');

// Input updates signal → greeting re-renders
return () => html`
  <div>
    <input oninput=${(e: Event) => setName((e.target as HTMLInputElement).value)} />
    <p>Hello, ${name()}!</p>
  </div>
`;
globalSignal() — shared state LIVE 
// Same key = same signal everywhere
const [theme, setTheme] =
  globalSignal('theme', 'light');

// ComponentA and ComponentB share it
setTheme('dark'); // both re-render
Component model

Components & Lifecycle

component(setup) runs setup once. Setup returns a render function that re-runs when its signals change. Handlers, derived state, and lifecycle hooks live in setup — created once, never re-created.

component() — todo list LIVE 
const Todo = component(() => {
  const [items, setItems] = signal([]);

  // handlers created once in setup
  const add = text =>
    setItems(it => [...it, { text, done: false }]);

  return () => /* render with items() */;
});
onInit / onMount / onDestroy LIVE 
const C = component(() => {
  // server + client
  onInit(() => log('init'));
  // client only — DOM ready
  onMount(() => log('mount'));
  // client only — cleanup
  onDestroy(() => log('destroy'));
  return () => html`<div>Hello</div>`;
});
ref() — DOM access LIVE 
const C = component(() => {
  const inputRef = ref();

  onMount(() => {
    // DOM is ready — .current is populated
    inputRef.current.focus();
  });

  return () => html`
    <input ref=${inputRef} placeholder="..." />
  `;
});
el.* — no-build hyperscript LIVE 
// el.* is a Proxy over h()
// props auto-detected — no null needed
el.div(
  el.h2('Hello'),
  el.p({ class: 'note' }, 'Works in script tags'),
  el.button({ onclick: fn }, 'Click me'),
)
Client-side routing

Built-in Router

createRouter() returns a component that re-renders reactively on navigation. Route params are signals. Zero dependencies.

createRouter() + navigate() + Link + params() LIVE 
const App = createRouter([
  { path: '/',           component: Home },
  { path: '/blog/:slug', component: BlogPost },
  { path: '*',           component: NotFound },
]);

// Programmatic navigation
navigate('/blog/hello-world');

// Declarative — prevents full reload
Link({ href: '/about', children: 'About' });

// Reactive param access
const p = params(); // { slug: 'hello-world' }

// Query string
navigate('/search?q=actjs');
const q = query(); // { q: 'actjs' }
Async data

resource() + Suspense

Async data loading with built-in loading and error states. On the server, data is awaited before streaming. On the client, the component re-renders when data arrives.

resource() — async fetch LIVE 
const posts = resource(() =>
  fetch('/api/posts').then(r => r.json())
);

return () => posts.loading()
  ? html`<div>Loading...</div>`
  : posts.error()
  ? html`<p>Error!</p>`
  : html`<ul>${posts().map(renderPost)}</ul>`;
Suspense — streaming boundary SSR
On the server, Suspense streams fallback HTML immediately, then swaps to the real content as async data resolves.
SSR stream
→ streams immediately:
<div class="skeleton">Loading...</div>
→ swaps when resolved:
<PostList posts={[...]} />
// SSR: streams fallback immediately,
// swaps content when data resolves
<Suspense fallback={<div>Loading...</div>}>
  <PostList posts={posts} />
</Suspense>
Island architecture

SSR + Island Hydration

Each component declares its own hydration strategy. Static components ship zero JS. Interactive islands hydrate on load. Visible islands hydrate when scrolled into view.

Static

Renders to pure HTML on the server. Zero JavaScript shipped. Perfect for headers, footers, marketing copy.

hydrate: 'static'

Interactive

Hydrates immediately on page load. Best for above-the-fold widgets that need to be interactive right away.

hydrate: 'interactive'

Visible

Hydrates only when scrolled into view using IntersectionObserver. Best for LCP — skips JS for off-screen content.

hydrate: 'visible'

Use renderToString(component) from actjs/server for SSR. On the client, pass { hydrate: true } to createApp() to adopt the server-rendered DOM without clearing it — bots see full HTML, users get a reactive island.

SSR usage SERVER 
// actjs/server subpath
import { renderToString } from 'actjs/server';

const html = await renderToString(MyPage);
res.send(`<!DOCTYPE html>
<html><body>${html}</body></html>`);
Client hydration CLIENT 
// Adopt SSR DOM — don't clear it
createApp('#root', { hydrate: true })
  .mount(MyPage);

// static island — zero JS shipped
const Header = component(() => () => ...,
  { hydrate: 'static' }
);
Template options

JSX, el.*, and html``

Three ways to write templates — pick what fits your stack. All produce real DOM nodes directly. No virtual DOM involved.

JSX (TypeScript) BUILD 
// tsconfig.json
"jsx": "react-jsx",
"jsxImportSource": "actjs"

// component
return () => (
  <div class="card">
    <h2>{title()}</h2>
    <p>{body}</p>
  </div>
);
el.* (no build) IIFE 
// Proxy over h() — works in
// plain <script> tags
return () =>
  el.div({ class: 'card' },
    el.h2(title()),
    el.p(body),
  );
html`` (tagged template) ESM 
// Tagged template → Element (no build)
// Functions wire as event listeners
return () => html`
  <div class="card">
    <h2>${title()}</h2>
    <button onclick=${fn}>
      Click
    </button>
  </div>
`;
SEO

useHead — SSR-aware Head Management

Set document title and meta tags from inside any component. On the server they're injected into the HTML stream. On the client they update <head> in place.

useHead() SSR + CLIENT 
const BlogPost = component(() => {
  const [title, setTitle] = signal('My Post');

  // Reactive — updates <title> when title() changes
  useHead({
    title: `${title()} | My Blog`,
    meta: [
      { name: 'description', content: 'Post summary...' },
      { property: 'og:title', content: title() },
    ],
    link: [
      { rel: 'canonical', href: 'https://example.com/blog/my-post' },
    ],
  });

  return () => html`<article>${title()}</article>`;
});
App lifecycle

createApp() — App Orchestrator

Mount components, inject CSS, run safe timers that auto-clear on destroy. Multiple independent apps on one page — each is isolated.

createApp() API 
const app = createApp('#root');

// Mount component
app.mount(Counter);

// Inject critical CSS (removed on destroy)
app.criticalCSS('.btn { color: red }');
app.criticalStylesheet('/app.css');

// Safe timers — auto-cleared on destroy
app.safeSetInterval(tick, 1000);

// Cleanup everything
app.destroy();
Multiple islands per page PATTERN 
// Each island is independent
createApp('#cart').mount(Cart);
createApp('#search').mount(SearchBar);
createApp('#newsletter').mount(Newsletter);

// SSR hydration — adopt server DOM
createApp('#root', { hydrate: true })
  .mount(App);
