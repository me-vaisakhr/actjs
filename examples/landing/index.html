01 Philosophy

Built on three principles.

Every design decision in actjs traces back to one of three principles. No magic, no hidden runtime cost, no framework lock-in.

⚡

Explicit Subscriptions

Calling a signal getter inside a computed or render function creates a live subscription. No decorators, no proxies, no compiler magic — just a function call. You always know exactly what re-renders and why.

// only re-renders when count() changes const doubled = computed(() => count() * 2);
🏝

Island Architecture

Static components render on the server and ship zero JavaScript. Interactive islands hydrate on demand. The page loads as fast as plain HTML — JS only enters where you explicitly call for it.

// zero JS shipped for static components component(setup, { hydrate: 'static' }); component(setup, { hydrate: 'interactive' });
📦

No Build Required

Drop a single script tag and start building. When you're ready for TypeScript, JSX, and a build step, actjs meets you there too — same API, same mental model, zero friction either way.

// CDN, no build step <script src="https://cdn.actjs.dev/actjs.js"></script>

06 External Packages

CDN deps, declarative and zero-config.

Declare external packages in actjs.deps.json. The CLI resolves versions from npm, the Vite plugin injects import maps at build time, and useDeps() loads them at runtime for script-tag users — all from the same config file.

actjs add — declare deps cli
# resolves latest version from npm registry actjs add lodash actjs add bootstrap@5.3.3 // actjs.deps.json (auto-generated) { "provider": "esm.sh", "packages": { "lodash": "4.17.21", "bootstrap": "5.3.3" } }
actjsPlugin() — build-time vite
// vite.config.ts import { actjsPlugin } from 'actjs/vite' export default defineConfig({ plugins: [actjsPlugin()], }) // then import by bare name — no URL needed import _ from 'lodash' // ↑ resolved via injected import map
useDeps() — runtime (IIFE) live demo
CDN providers config
// esm.sh (default) — ESM + tree-shaking "provider": "esm.sh" // → https://esm.sh/lodash@4.17.21 // jsDelivr "provider": "jsdelivr" // → cdn.jsdelivr.net/npm/lodash@4.17.21/+esm // unpkg "provider": "unpkg" // → unpkg.com/lodash@4.17.21?module

07 Island Architecture

Pay only for what you ship.

Three hydration modes let you choose the cost of each component precisely. Static islands never touch the JS bundle. Interactive islands hydrate eagerly. Visible islands defer until they enter the viewport.

static

Zero JS

Rendered on the server. No JavaScript shipped to the client. Ideal for navigation, footers, and any content that doesn't change.

component(setup, { hydrate: 'static' }); // → 0 bytes JS
interactive

Eager Hydration

Hydrates immediately after page load. Full signal reactivity available. Use for above-the-fold interactive elements where time-to-interactive matters.

component(setup, { hydrate: 'interactive' }); // → hydrates on load
visible

Deferred Hydration

Uses IntersectionObserver to hydrate only when the component enters the viewport. Best for below-the-fold content — keeps TTI minimal.

component(setup, { hydrate: 'visible' }); // → hydrates on scroll
08 Framework Compatibility

Works with any backend.

actjs is framework-agnostic. Render the shell server-side in Django, Express, Rails, or anything that outputs HTML — then mount islands on the client.

🐍 Django / Jinja2
{# templates/index.html #} <!-- Static shell rendered by Django --> <div class="hero"> <h1>{{ page.title }}</h1> </div> <!-- Interactive island --> <div id="cart-widget"></div> <script type="module" src="/static/app.js"></script> // app.js import { createApp } from 'actjs'; import { CartWidget } from './CartWidget'; createApp('#cart-widget').mount(CartWidget);
🟩 Node.js / Express
// server.js — SSR shell import { renderToString } from 'actjs/server'; import { App } from './App'; app.get('/', async (req, res) => { const html = await renderToString(App, { props: { user: req.user } }); res.send(`<!DOCTYPE html> <html><body> <div id="root">${html}</div> <script type="module" src="/app.js"></script> </body></html>`); });
09 API Reference

The complete surface area.

Small, intentional, and stable. Everything actjs exports — no surprises.

Name Signature Description Category
signal signal<T>(init: T) → [() → T, (v: T | (prev: T) → T) → void] Creates a reactive value. Getter subscribes; setter triggers updates. core
computed computed<T>(fn: () → T) → () → T Derives a memoised value from other signals. Re-evaluates only when dependencies change. core
globalSignal globalSignal<T>(key: string, init: T) → [() → T, setter] Application-scoped signal shared across component instances. core
component component(setup: () → renderFn, opts?: Options) → Component Defines a component. Setup runs once; returned render function runs reactively. core
createApp createApp(selector: string) → { mount(c: Component): void } Mounts a component into a DOM target. Entry point for island hydration. core
onInit onInit(fn: () → void) → void Runs during component setup — executes on both server and client. lifecycle
onMount onMount(fn: () → void | (() → void)) → void Runs after the component is inserted into the DOM. Return a cleanup function. lifecycle
onDestroy onDestroy(fn: () → void) → void Runs when the component is removed from the DOM. lifecycle
ref ref<T extends Element>() → { current: T | undefined } Creates a mutable reference to a DOM element. Populated after mount. dom
el.* el.div(attrs?, ...children) → HTMLElement Programmatic element builders — all HTML tags available as methods. dom
textBind textBind(fn: () → string) → Text Creates a live text node that updates without re-rendering parent elements. dom
createRouter createRouter(routes: Route[]) → Router Hash or history-based router. Exposes current path and params as signals. routing
resource resource<T>(fetcher: () → Promise<T>) → ResourceState<T> Wraps an async data source. Exposes loading, error, and data signals. async
renderToString renderToString(c: Component, opts?) → Promise<string> Server-side rendering entry point. Returns the HTML shell as a string. async
